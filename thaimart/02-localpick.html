กรณีศึกษา

🛵ThaiMart LocalPick

ลูกค้าออนไลน์เห็น สต็อกร้านใกล้บ้านแบบเรียลไทม์ และจองสินค้าเพื่อ รับถึงรถภายใน 2 ชั่วโมง — เปลี่ยนสต็อกบนชั้นวางที่ไม่ได้ขายให้กลายเป็นออร์เดอร์ที่ได้ของทันใจ โดยไม่ต้องใช้ไรเดอร์ส่งของ

ปัญหา

ThaiMart สัญญาว่าจัดส่ง "ภายในวันเดียว" แต่ลูกค้ากรุงเทพฯ ที่อยากได้เบียร์แพ็ก 6 เดี๋ยวนี้ ไม่รอ 4–6 ชั่วโมง — พวกเขาเดินไปเซเว่นแทน ขณะเดียวกันร้านนำร่องทุกสาขาต่างมีสินค้าจริงนับพันรายการบนชั้นวาง ที่แอปอีคอมเมิร์ซมองไม่เห็น ความต้องการที่รวดเร็วกับสินค้าที่ยังไม่ถูกขายอยู่ห่างกันแค่หนึ่งกิโลเมตร แต่ไม่เคยมาเจอกัน

MVP / ขอบเขต — การจัดลำดับความสำคัญแบบ Agile

เปิดตัวที่ ร้านนำร่องเพียงสาขาเดียว ใช้สิ่งที่มีอยู่ พิสูจน์ว่ากระบวนการครบวงจรได้ผล แล้วค่อยขยายทีละสาขา

✅ อยู่ในขอบเขต (MVP)

  • ร้านนำร่องหนึ่งสาขา (ไทยมาร์ท สุขุมวิท 39)
  • ใช้ inventory API เดิม — อ่านสต็อก + กันสินค้าไว้ให้ (หมดอายุใน 2 ชม.)
  • แอปจัดสินค้าสำหรับพนักงานบนแท็บเล็ตที่มีอยู่
  • การจองสินค้าชิ้นเดียว / ตะกร้าขนาดเล็ก
  • ส่งมอบแบบรับถึงรถ จ่ายตอนรับสินค้า
  • สถานะออร์เดอร์: จองแล้ว → กำลังจัดสินค้า → พร้อมให้รับแล้ว → รับสินค้าแล้ว

⛔ นอกขอบเขต (ภายหลัง)

  • ฮาร์ดแวร์ใหม่ (ตู้ล็อกเกอร์ เครื่องสแกน อุปกรณ์เฉพาะทาง)
  • การชำระเงินล่วงหน้าและคืนเงินในแอป
  • ตะกร้าข้ามร้าน / การปรับสมดุลสต็อก
  • การติดตามเวลามาถึงแบบเรียลไทม์และ GPS geofencing
  • การจัดเส้นทางพนักงานอัตโนมัติ / การจัดสินค้าแบบเป็นชุด
  • การขยายทั่วประเทศก่อนบรรลุตัวชี้วัดของโครงการนำร่อง
