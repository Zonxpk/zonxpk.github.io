    ขอบเขต MVP

    เปิดตัวเร็วด้วยการใช้ของเดิมที่มีอยู่

    • ใช้ของเดิม Identity / KYC API — การยืนยันเลขประจำตัวผู้เสียภาษีและตรวจเอกสารใช้ในระบบ KYC ของผู้ซื้ออยู่แล้ว ไม่ต้องสร้างระบบ compliance ใหม่
    • ใช้ของเดิม Catalog API — ใช้ endpoint เผยแพร่ตัวเดียวกับที่ทีมปฏิบัติการใช้อยู่ทุกวันนี้ pipeline แค่เรียกใช้แบบอัตโนมัติ
    • ใช้ของเดิม บริการรับเงิน — การยืนยันบัญชีธนาคารและการโอนเงินคงเดิม

    ทำอัตโนมัติ vs ยังต้องใช้คนตรวจ

    • อัตโนมัติ การอ่านไฟล์ CSV จับคู่คอลัมน์ กำหนดหมวดหมู่ด้วย AI และกฎตรวจสอบ
    • อัตโนมัติ เผยแพร่รายการที่ความมั่นใจ ≥ 90% เข้าสู่ระบบสินค้าโดยตรง
    • ใช้คนตรวจ เฉพาะรายการที่ถูกทำเครื่องหมาย ต้องตรวจสอบ (ความมั่นใจต่ำ / หมวดหมู่ที่จำกัด) จึงจะถึงมือเจ้าหน้าที่ปฏิบัติการ — ราว 10% ของทั้งหมด เทียบกับ 100% ในปัจจุบัน
    • ใช้คนตรวจ กรณีพิเศษของ KYC สำหรับผู้ขายรายใหม่

    นอกขอบเขต MVP

    การแปลรายการสินค้าหลายภาษา การติดแท็กรูปภาพอัตโนมัติ และการซิงก์กับ POS แบบเรียลไทม์ เป็นงานที่จะตามมาทีหลัง v1 พิสูจน์ผลด้านเวลาจนเปิดร้านจากการนำเข้าไฟล์ CSV เพียงไฟล์เดียว
