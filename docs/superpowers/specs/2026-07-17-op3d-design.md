# OP-3D — Teenage Engineering 3D Elements for op3d.html

Date: 2026-07-17
Status: Approved (brainstormed interactively; decisions recorded below)
Companion to: `2026-07-17-op-redesign-design.md` (Lab White base design), `OP-3d.md` (3D brief)

## Goal

Build `op3d.html`, a sibling of the redesigned `index.html`, carrying the same
content and Lab White layout but upgraded with interactive Three.js 3D hardware
in the "Playful Laboratory" mood defined by `OP-3d.md`: dense, clinical,
tactile — an interactive technical blueprint, not a glossy product render.

## Decisions

1. **Target:** New standalone page `op3d.html`. `index.html` and `grid.html`
   untouched. Promotion to index is a separate later decision.
2. **Stack:** Three.js, pinned version, loaded from a CDN via import map.
   This intentionally relaxes the base design's zero-dependency rule for this
   page only. No other dependencies, no webfonts, no build step.
3. **Form:** 3D sprinkled throughout — a hero gadget plus one 3D widget per
   module (see element map). Content, copy, and module structure are copied
   from `index.html` unchanged.
4. **Micro-interactions in scope:** rotary detents (15° with micro-shudder),
   film grain overlay, stiff spring key physics.
   **Out of scope:** cursor-flashlight shadow tracking (rejected — forces a
   continuous render loop).
5. **Mobile/fallback:** full 3D everywhere including phones (drag-to-turn).
   If the CDN import or WebGL context fails, placeholders keep their inline
   SVG art and the page stays fully usable with zero console errors.
6. **Canvas architecture:** one shared WebGL context, scissored viewports
   (the standard three.js "multiple elements" pattern).

## Visual system (3D)

- **Camera:** orthographic, isometric-style angle. No perspective distortion.
- **Lighting:** clinical — soft white ambient plus one large diffused top
  directional light casting crisp shadows onto a light grid plane. Static; no
  cursor-tracking light.
- **Materials** (from `OP-3d.md`):
  - Chassis: sandblasted anodized aluminum — metalness 0.9, roughness 0.5,
    fine-grain normal texture.
  - Keys: chalky injection-molded plastic — metalness 0, roughness 0.9, flat
    toy colors reusing the page tokens (`--c-orange` etc.).
  - Knobs: ribbed dark rubber — geometric ridges (real geometry or normal
    map), low specular.
  - Screen: recessed matte glass over an emissive dot-matrix texture; the
    matrix displays the current section code.
- **Palette discipline:** identical tokens to `index.html`; accents only on
  interactive 3D parts, chassis stays neutral aluminum/white.

## Interaction physics

- **Keys:** stiff spring — compress instantly on press, pop back with sharp
  high-frequency recoil. Wired into the existing SND square-wave click
  (same opt-in toggle, same `localStorage['snd']`).
- **Knobs:** 15° detents with a micro-shudder on each snap. Input: drag to
  turn always; wheel-to-turn only while the pointer is over the knob
  placeholder (never steals page scroll elsewhere).
- **Fader:** drag; position quantizes to grid increments, no easing/glide.
- **Film grain:** page-wide overlay div — SVG `feTurbulence` noise, animated
  with `steps()` frame-jumps, very low opacity, `pointer-events: none`.

## Architecture

- **One canvas** fixed full-viewport behind a `pointer-events: none` style;
  each 3D element is a normal DOM placeholder in the page flow that receives
  pointer/keyboard input. Per frame, the renderer scissors and draws each
  visible element's scene into its placeholder's bounding rect.
- **On-demand rendering:** dirty-flag per element; a single rAF loop runs only
  while an animation (spring, detent shudder) is active or a placeholder both
  intersects the viewport and is dirty (scroll/resize mark visible elements
  dirty). Idle page performs no GPU work. The CSS grain overlay is the only
  always-running animation.
- **Loading:** `<script type="importmap">` pinning the three.js CDN URL, then
  one `<script type="module">`. Failure of either (network, no WebGL) is
  caught and leaves the SVG fallbacks in place.
- **Fallbacks:** each placeholder ships with its inline SVG art in the HTML
  (hero: the existing signal-path schematic; rails: the existing color chips;
  LAB tiles: the existing glyphs). On successful WebGL boot, the SVG is
  hidden and the 3D view takes over. No layout shift between states.
- **Accessibility:** every 3D control mirrors an existing accessible pathway
  (keys ↔ nav links/QWER, encoder ↔ table rows already readable, toggle is
  decorative). Placeholders get `aria-hidden` where purely decorative and
  real button semantics where interactive.

## Out of scope

- Cursor-flashlight shadow shifting.
- Restyling `index.html`, `grid.html`, or sub-project pages.
- Linking op3d.html from index.html (separate decision, as with grid.html).
- Dark theme; audio beyond the existing synthesized click.

## Verification

Playwright against `python3 -m http.server 8000`:

1. Desktop 1280/1680 and 390 viewport: `scrollWidth` ≤ viewport, no overflow.
2. Zero console errors with WebGL available.
3. Block the three.js CDN request → page renders SVG fallbacks, zero console
   errors, all links still work.
4. Drive behaviors: hero key press scrolls to its module and updates the
   readout; knob drag steps in detents; SND toggle persists across reload.
5. All local prototype links return 200.
